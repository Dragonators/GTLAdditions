# ME Super Pattern Buffer

<Column gap="15" fullWidth={true}>

<Row>
    <BlockImage id="gtladditions:me_super_pattern_buffer" scale="6" />
    <BlockImage id="gtladditions:me_super_pattern_buffer_proxy" scale="6" />
</Row>

<Column gap="2" fullWidth={true}>

* You need to reach the <Color color="#FFFF00">**UXV**</Color> stage before you can craft it.
* ME Super Pattern Buffer is a larger ME Pattern Buffer with configurable patterns per row, rows per page, and maximum pages.
* It provides every capability of <ItemLink id="gtceu:me_extend_pattern_buffer" />.
* GUI pagination is controlled by the configuration, making it suitable for centralized management of many terminal patterns.
* It also supports GTLCore's <ItemLink id="gtlcore:me_pattern_buffer_copy" /> and <ItemLink id="gtlcore:me_pattern_buffer_cut" />.

</Column>

<Column gap="2" fullWidth={true}>

## Flash Drive Binding

* Right-click <ItemLink id="gtladditions:me_super_pattern_buffer" /> while holding a <ItemLink id="gtceu:data_stick" /> to record that buffer's binding information.
* Right-click <ItemLink id="gtladditions:me_super_pattern_buffer_proxy" /> while holding a <ItemLink id="gtceu:data_stick" /> that already has binding information to bind the proxy to the matching <ItemLink id="gtladditions:me_super_pattern_buffer" />.
* The proxy only accepts coordinates for <ItemLink id="gtladditions:me_super_pattern_buffer" />. Normal Pattern Buffer coordinates are not valid targets.
