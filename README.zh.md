# GTLAdditions

[English](README.md)

> [!TIP]
> GTLAdditions 是面向 GregTech Leisure 环境的扩展模组，围绕 GTCEu 与 GTLCore 添加后期多方块、机器仓室、配方、材料、渲染器、GuideME 文档与外部模组联动。
>
> 精确配方与结构预览请以 JEI 和游戏内 GTLAdditions 指南为准。

## 前言

GTLAdditions 基于 Minecraft Forge 1.20.1 与 GregTech Leisure 生态开发。在保留原版硬核科技框架的同时，扩展了 GTCEu/GTLCore 的后期流程，加入大量创新设计与平衡性优化，为玩家带来更流畅的自动化体验，以及一个更高难度的终极目标。

## 前置需求

| 需求 | 版本 / 范围 |
| --- | --- |
| GTLCore (`gtlcore`) | `>= 1.2.3.0` |
| Stargate Journey (`sgjourney`) | `>= 0.6.44` |
| Oculus (`oculus`) | `>= 1.7.0`，客户端侧 |

## 安装流程

1. 在[此处](https://pan.quark.cn/s/d13f899cdab5#/list/share)或[这里](https://drive.google.com/drive/folders/1Ga_w-TmDKNru0me1kAM_gXyedz_Ne4-x)下载 `GregTech Leisure`，然后使用你的 Minecraft 启动器安装它。
2. 删除其中的 `GTLCore`，接着在[这里](https://github.com/AaAdoniSsS/GTLCore)下载并安装最新版 GTLCore。
3. 将 GTLAdditions 添加到 `mods` 文件夹中。
4. 使用 JEI 和游戏内 GTLAdditions 指南查询配方、结构与功能细节。

### 其他修改内容

- `无线光学数据仓` 和 `无线算力数据仓` 可被多方块结构共用。
- `可配置自动维护仓` 系列可被多方块结构共用，同时可在内部放入物品加强仓室效果。

### 配方

- 添加若干个配方以简化后期的大型产线。请学会使用 **`JEI`** 来查找配方。如果某个配方来自 `GTLAdditions`，JEI 中会有注解。多查看 JEI，也许你能发现新的 GTLAdditions 配方。
- 新添加的 SoC 产线为后期电路板添加了高效配方。
- 游戏内 GTLAdditions 指南的魔改页面也会介绍部分独特的后期流程配方。

如需更多信息，请查阅游戏内的 GTLAdditions 指南。


## 构建 / 开发

```bash
# 构建模组
./gradlew build

# 启动开发客户端
./gradlew runClient

# 启动开发服务端
./gradlew runServer

# 生成数据
./gradlew runData

# 检查格式
./gradlew spotlessCheck

# 应用格式
./gradlew spotlessApply
```

如果当前检出没有 Gradle wrapper，请改用 `gradle` 执行对应任务。
